Plan de formation IA, Automatisation & Vibe Coding

Formateur : Lorenz Campan, Consultant IA & Automatisation
Format : Sessions individuelles de 1h30 à 2h, présentiel ou visio selon nos agendas

Ce parcours n'est pas un cours de code. C'est un accompagnement pour que tu sois autonome sur ton projet : prototyper, automatiser et déployer sans faire appel à un développeur.

L'ordre est pensé pour que chaque bloc prépare le suivant : d'abord comprendre vraiment comment l'IA fonctionne, ensuite automatiser des tâches avec elle, enfin construire ton app. À chaque étape, on alterne théorie et pratique, avec des cas concrets.

Module 1 : IA Générative & Fondamentaux

4 sessions · environ 6 à 7h

On commence par comprendre ce que l'IA est vraiment, sans le marketing.

Session 1 : Comment ça marche
Les bases techniques vulgarisées : machine learning, LLMs, transformers, tokens, fenêtre de contexte. Et les limites qu'il faut connaître : hallucinations, biais, RGPD. Ces fondamentaux ne sont pas de la culture générale : c'est ce qui te permettra de construire des systèmes fiables dans les modules suivants.
Session 2 : Panorama des outils & choisir le bon modèle
Tour des grandes catégories d'IA (texte, image, vidéo, son) avec mise en pratique sur chacune. Et surtout : comment choisir le bon modèle selon la tâche, avec les critères qui comptent vraiment (raisonnement, contexte, coût, confidentialité).
Session 3 : Prompt Engineering
La compétence clé. Structure d'un prompt efficace, few-shot, chain of thought, structured output, system prompts. Et les paramètres des modèles (température, top P…) pour affiner les résultats. On s'entraîne sur des cas concrets.
Session 4 : Assistants IA, projets & RAG
Créer un assistant IA personnalisé et persistant avec Claude Projects ou ChatGPT. Introduction au RAG (Retrieval Augmented Generation) : comment donner à l'IA accès à tes propres documents (CVs, fiches de poste, contrats) sans l'entraîner. Directement pertinent pour ton app.

Module 2 : Automatisation

6 sessions · environ 8 à 9h

Faire travailler les outils à ta place sur tout ce qui est répétitif.

Session 5 : Introduction
La logique d'une automatisation (trigger → condition → action), panorama des outils (Make, n8n, Zapier), premier cas pratique reproductible en live.
Session 6 : Make
Prise en main complète : scénarios, modules, routes, itérateurs, gestion des erreurs. Cas pratique fil rouge.
Session 7 : API, Webhook, HTTP, MCP
Les concepts derrière tous les outils d'automatisation, expliqués sans jargon. Mise en pratique avec le HTTP node pour appeler n'importe quelle API sans coder.
Session 8 : n8n
L'outil d'automatisation avancé. Interface, nœuds essentiels, expressions, connexion de services. Gestion des coûts quand on appelle des modèles IA depuis un workflow.
Session 9 : Agents IA
La différence entre une automatisation classique et un agent qui prend des décisions. Construction d'agents dans n8n : outils, mémoire, cas pratiques concrets.
Session 10 : Cas pratiques & panorama no-code
Consolidation avec des workflows réels. Tour rapide d'un ou deux outils no-code pour avoir le panorama complet.

Module 3 : Vibe Coding

8 sessions · environ 11 à 12h

Prototyper et déployer ton app avec l'IA comme co-développeur.

Session 11 : Introduction & panorama des outils
Philosophie du vibe coding et boucle de travail. Tour des principales plateformes de vibe coding : Lovable, Bolt, Base44, v0, Replit, Cursor. L'objectif est d'avoir le panorama et de savoir vers quoi se tourner selon son projet et son niveau.
Session 12 : Fondamentaux coding
Pas pour apprendre à coder, mais pour comprendre à quoi sert chaque couche. HTML, CSS, JavaScript, React, frontend/backend, Supabase, GitHub, hébergement. Ce qu'il faut savoir pour diriger l'IA sans être dans le flou.
Session 13 : Specs & PRD
Avant de toucher au code : rédiger une spec solide. Structure d'un bon PRD, cas pratique sur ton projet, et comment la spec devient le point de départ de tout le développement.
Session 14 : Écosystème agentique
Claude Code, Codex, Cowork, harnais agentiques et leur logique commune. Les fichiers .md à la racine d'un projet, les skills, les plugins, la gestion du contexte et de la mémoire. Ce qui fait passer du vibe coding itératif au vibe coding en autopilote.
Session 15 : Vibe coding avancé
Refactoring et nettoyage du code avec des skills dédiés. Agent teams et parallélisation. Connexion des workflows n8n à l'app.
Session 16 : Sécurité
Les réflexes incontournables : variables d'environnement, .gitignore, gestion des clés, authentification, RGPD basique. Comment auditer la sécurité d'un projet vibe-codé.
Sessions 17 & 18 : Cas pratiques & déploiement
Construction d'une fonctionnalité concrète du projet en live, connexion des automatisations, déploiement GitHub → Vercel. Environnements dev/prod, tests manuels. Bilan et plan d'autonomie pour la suite.

Phases de coaching projet

En parallèle ou en suite des modules, des sessions dédiées à ton projet : déblocage, review, validation d'architecture, préparation d'une démo. Format et rythme à définir selon les besoins.
